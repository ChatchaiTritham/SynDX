## 🚨 Read This First

**This is PhD research code—not production medical software.** We've only tested this on synthetic data (fake patients generated by the system itself). We haven't validated it against real patient outcomes, so **do NOT use this for actual clinical decisions**. That would require proper clinical trials first.

## What Is This?

SynDX generates realistic synthetic medical data for vestibular (dizziness/vertigo) disorder research. The cool part? It doesn't need any real patient records to work. Instead, it learns from clinical guidelines and uses explainable AI to create privacy-safe synthetic patients.

Think of it as: Clinical guidelines → AI magic → Realistic fake patients for research

### Why You Might Care

- **Privacy-first**: No real patient data required, ever
- **Built on clinical knowledge**: Uses TiTrATE framework and Bárány Society guidelines
- **Explainable**: SHAP, counterfactuals, and interpretable models throughout
- **Standards-compliant**: Outputs HL7 FHIR, SNOMED CT, LOINC—all the healthcare acronyms
- **Actually reproducible**: Docker containers, example notebooks, the works

## How It Works (The 10,000-Foot View)

We built this in three phases:

**Phase 1: Extract Clinical Knowledge**

We take clinical guidelines (like TiTrATE for dizziness diagnosis) and formalize them into 8,400 computational "archetypes"—basically template patients that capture all the clinical decision rules.

**Phase 2: Generate Synthetic Patients**

Here's where the AI comes in:
- NMF finds latent patterns across archetypes
- VAE learns the underlying distribution
- SHAP ensures features make clinical sense
- Counterfactual validation checks that small changes behave realistically
- Differential privacy adds just enough noise to prevent memorization

**Phase 3: Validate Everything**

We test three ways:
- Statistical realism (do the numbers look right?)
- Diagnostic performance (can classifiers use this data?)
- XAI fidelity (do explanations match clinical reasoning?)

## Quick Example: Generate 10K Patients

Here's the gist:

```python
from syndx import SynDXPipeline

# Set it up
pipeline = SynDXPipeline(
    n_archetypes=8400,      # How many clinical templates
    nmf_components=20,       # Latent factors to extract
    vae_latent_dim=50,      # VAE bottleneck size
    epsilon=1.0             # Privacy budget (lower = more private)
)

# Phase 1: Pull knowledge from guidelines
archetypes = pipeline.extract_archetypes(
    guidelines=['titrate', 'barany_icvd_2025']
)

# Phase 2: Make some fake patients
patients = pipeline.generate(
    n_patients=10000,
    convergence_threshold=0.05
)

# Phase 3: Check if they look realistic
results = pipeline.validate(
    patients,
    metrics=['statistical', 'diagnostic', 'xai']
)

# Export to FHIR if you need interoperability
pipeline.export_fhir(patients, 'outputs/synthetic_patients.json')
```

## Performance (Remember: Synthetic-Only!)

⚠️ **Big caveat**: These numbers are from testing synthetic data against *other* synthetic data. They show internal consistency, not real-world performance.

**Statistical Realism**
- KL Divergence: 0.042 ± 0.004 (pretty close to the archetypes)
- Jensen-Shannon: 0.031 ± 0.003
- Wasserstein: 0.053 ± 0.005

**Diagnostic Performance** (on synthetic test set)
- ROC-AUC: 0.89 ± 0.02
- Sensitivity: 0.91 ± 0.03
- Specificity: 0.86 ± 0.03

**XAI Fidelity** (do the explanations make sense?)
- SHAP Fidelity: 91.6% (95% CI: 88.9-94.3%)
- TiTrATE Coverage: 94.2% (matches clinical guidelines)

## Limitations (The Reality Check)

### The Big One: No Real Patient Validation

Let's be blunt: **we haven't tested this on actual patients**. All our numbers come from synthetic-to-synthetic comparisons. That means we know the internal math is consistent, but we don't know if it generalizes to real emergency departments.

Before anyone should use this clinically, we need:
- Validation on 500-1,000 real ED patients
- Multi-center trials across different hospitals
- FDA clearance (510(k) pathway, probably)

### Other Things to Know

- **Domain-specific**: Built for vestibular disorders. Other conditions would need adaptation.
- **Compute-heavy**: GPU recommended. We developed on an A100.
- **Guidelines freeze**: Based on TiTrATE 2015 and Bárány ICVD 2025. Medicine evolves.

## What This Is Good For (And What It Isn't)

### ✅ Good Uses (No Real Patients Required)

- Prototyping diagnostic algorithms
- Creating benchmarks for ML models
- Testing privacy-preserving techniques
- Educational demos
- Reproducible research comparisons

### ❌ Bad Uses (Need Clinical Validation First)

- Making actual clinical decisions
- Diagnosing real patients
- Submitting to regulators
- Enrolling patients in trials

Basically: research and development? Great. Clinical care? Not yet.
