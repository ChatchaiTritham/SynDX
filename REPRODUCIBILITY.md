# Reproducibility

This document states honestly what the committed code reproduces.

## How to run

```bash
git clone https://github.com/ChatchaiTritham/SynDX.git
cd SynDX
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -e .
python -m pytest -q
```

The package is deterministic under a fixed seed (default seed 42 in the generation
and validation entry points).

## What reproduces from the committed code

- The full unit + integration test suite (`python -m pytest -q`): generation,
  differential-privacy accounting, counterfactual validation, diagnostic
  evaluation, and the end-to-end validation workflow.
- The manuscript figures listed in `FIGURE_MANIFEST.csv`, regenerated by the
  matching scripts under `scripts/` and `examples/` (SHAP importance, focused
  validation metrics, counterfactual quality profile).

## Scope and limitations

- SynDX generates and validates **synthetic** clinical records; no real patient
  data is included or required.
- Figure scripts read from the package's generation/validation outputs; numbers
  in the manuscript are produced by these committed routines at the stated seed.
